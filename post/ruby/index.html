Ruby有时会让人感觉出奇地像英语。Ruby只是数百种编程语言中的一种，但它很特别，因为对许多程序员来说，它感觉很像一种自然语言，同时又具有计算机所要求的清晰性。如果运行下面的代码，将会打印Hello, world!十次。

10.times do print "Hello, world!" end

在irb中，上面代码运行后除了会打印十次Hello, world!，还会出现=> 10，这表明10.times do print "Hello, world!" end作为表达式的结果为10，这是因为Ruby中的所有代码行都是由有返回值的表达式组成的。

print "test"
test => nil

上面的代码执行后会打印test，并且后面紧跟着=> nil。这是因为print除了打印，不会返回任何值，所以print "test"作为表达式的返回值为nil。

有经验的程序员可能想知道为什么上面的代码示例的末尾没有分号。与许多其他语言不同，例如C#、php、C和C++，Ruby中的行尾不需要分号（尽管使用分号也无妨）。一开始可能需要一点时间来适应，但对于新程序员来说，它使Ruby更容易学习。

puts和print的区别

puts和print的区别：puts输出内容后自动将输出光标移动到下一行。在下面的代码中，print输出后不会换行，所以使用puts输出时world将紧接着hello，然后换行，最后输出test：

print("hello ")
puts("world")
puts("test")
// hello world
// test

puts "x".ord
puts 120.chr

Basic Variables

  • 局部变量：局部变量以小写字母或下划线开头，由字母、下划线和/或数字组成。x、string、abc、var1、start_value 和 firstName都是有效的局部变量名。但是请注意，Ruby约定在组合多个单词的局部变量名称时使用下划线而不是驼峰式大小写。例如，一般使用first_name而不是firstName。
  • 实例变量：用于在单个对象中存储信息的实例变量始终以单个@符号开头，其后包含与局部变量相同的字符集，例如@age和@last_name。虽然局部变量不能以大写字母开头，但实例变量可以在@符号之后的第一个位置有一个（尽管它可能在这个位置没有数字）。但通常@符号后面的字符是小写字母。
  • 类变量：存储每个类层次结构的信息（同样，在这个阶段不要担心语义），遵循与实例变量相同的规则，除了它们以两个@@符号开头——例如，@@running_total。
  • 全局变量可以通过它们的前导美元符号来识别——例如，$population。美元符号后面的段不遵循局部变量命名约定，全局变量$:, $1, $/,$stdin,$LOAD_PATH都是合法的。只要它以美元符号开头，它就是一个全局变量。至于非字母数字标识符，你可能会看到的唯一此类标识符是预定义的，因此无需担心哪些标点符号是合法的，哪些不是。

Ruby有许多关键字–预先定义的，与特定的编程任务和环境相关的保留关键字。关键字包括def（用于方法定义）、class（用于类定义）、if（条件执行）和__FILE__（当前正在执行的文件的名称）。其中大约有40个，它们通常是简短的单字（而不是下划线组成）标识符。

在Ruby中，方法的名称遵循与局部变量相同的规则和惯例（除了它们可以以?、!或=结尾，其意义你将在后面看到）。这是设计好的：方法并不像方法那样引起人们的注意，而是作为提供值的表达式融入到程序的结构中。在某些情况下，你无法通过观察一个表达式来判断你看到的是一个局部变量还是一个方法名–这是故意的。

说到方法，现在你已经有了一个Ruby标识符的路线图，让我们回到一些语言语义上，特别是对象及其方法的重要作用。

Data, Expressions, and Flow Control

计算机程序几乎所有的时间都在操作数据或等待数据从其它地方到达。我们输入文字、短语和数字；听音乐；看视频，而计算机则进行计算，作出决定，并向我们传递信息。要编写计算机程序，必须了解数据及其操作的基本知识。

Numbers and Expressions

编程时，表达式是数据（例如数字或文本字符串）、运算符（例如 + 或 -）和变量的组合，当计算机理解时，这些变量会产生某种形式的答案。例如，这些都是表达式

5
1 + 2
"a" + "b" + "c"
100 - 5 * (2 - 1)
x + y

变量是占位符或对对象的引用，包括数字、文本或您选择创建的任何类型的对象。可以随心所欲地命名变量，只有一些限制。变量名称必须是一个单位（没有空格！）；必须以字母或下划线开头；必须只包含字母、数字或下划线；并且区分大小写。

变量名 是否有效
x 是
Y2 是
_x 是
7x 否，字母开头
this_is_a_test 是
this is a test 否，包含空格
this’is@a’test! 否，包含特殊字符‘，@，！
this-is-a-test 否，包含特殊字符-

由于变量是值和数据的占位符，因此它们也可以用表达式的结果赋值（例如x = 2 - 1）并用于表达式本身（例如 x - y + 2）。

没有逻辑的程序仅仅是一个计算器。计算机不只是对数据进行单一操作。它们还使用逻辑来决定不同的行动方案。逻辑的一个基本形式是在表达式中使用比较运算符来做决定:

age = 10 
puts "You're too young to use this system" if age < 18

age = 24 
puts "You're NOT a teenager" unless age > 12 && age < 20

Ruby提供了另一个可爱的技术是between?方法，如果对象介于或等于两个提供的值，则返回真或假。例如，至少在处理整数时，age.between?(13, 19)相当于age >= 13 &&age <= 19：

age = 24
puts "You're a teenager" if age.between?(12,20)

Looping Through Numbers with Blocks and Iterators

在下面的代码中，首先取了5，然后调用times方法（这个方法对于所有的数字都适用）。不是向这个方法传递数据，而是向它传递更多的代码：do和end之间的代码。然后，times方法连续使用该代码五次，产生前面的五行输出：

5.times do puts "Test" end

另一种写法是用花括号代替do和end。尽管鼓励do和end用于多行代码块，但花括号使代码在单行上更易于阅读。因此，下面的代码的执行后结果完全相同：

5.times { puts "Test"}

也就说说，对于单行代码建议使用花括号这种风格，但对较长的代码块将使用do和end。这是一个很好的习惯，因为这几乎是所有专业的Ruby开发者所遵循的风格。

在Ruby中，创建循环的一种机制称为迭代器。迭代器是逐个遍历项目列表的东西。还有其他迭代器可用于数字，例如：

1.upto(5) { ...code to loop here... }

10.downto(5) { ...code to loop here... }

0.step(50, 5) { ...code to loop here... }

不明显的是如何知道在每一步迭代的数字，以便可以在循环代码中使用它。如果想打印出当前的迭代次数怎么办？如何使用这些迭代器开发计数程序？值得庆幸的是，刚刚解释的所有迭代器都会自动将迭代的状态作为参数传递给循环代码，然后可以将其检索到变量中并使用，如下所示：

1.upto(5) do |number|
  puts number
end

这里要意识到的关键点是，有些方法会执行代码块并传递数据，然后你可以将其捕获到变量中。在上面的例子中，对整数可用的upto方法将每次迭代的值传递到代码块中，我们将其"captured"到变量数字中。

Floating Point Numbers

默认情况下，Ruby将任何没有浮点数（也称为小数点）的数字视为整数。当你说 10 / 3 时，你要求Ruby将两个整数相除，而Ruby会给出一个整数作为结果。因此。下面的代码运行后的结果为3，不是期望的3.333：

puts 10 / 3

如果将代码做一些小小的改动：

puts 10 / 3.0
puts 10.0 / 3
puts 10.0 / 3.0

则输出的结果将会是3.3333333333333335。Ruby现在正在处理Float类的数字对象，并返回一个Float，并提供期望的精度级别。

在某些情况下，无法控制传入的数字，但仍希望将它们视为浮点数。考虑一种情况，用户输入两个要除的数字，而这些数字需要一个精确的答案：

x = 10
y = 3
puts x / y

上面代码中两个输入数字都是整数，所以结果是一个整数。幸运的是，整数有一种特殊的方法可以即时将它们转换为浮点数，只需像下面这样重写代码即可：

x = 10
y = 3
puts x.to_f / y

相似地，浮点数也可以转换为整数，只要使用to_i方法即可：

puts(5.7.to_i)
puts(5.7.to_int)

Constants

虑一些永远不会改变的值——例如pi的值。这些不变的值称为常量，在Ruby中也可以用以大写字母开头的变量名来表示：

Pi = 3.141592

如果再尝试改变Pi的值，虽然是可以的，但编译器会产生警告already initialized constant Pi。Ruby让我们可以完全控制常量的值，但警告会发出明确的信息。将来，Ruby可能会对常量实施更严格的控制，因此请尊重这种使用方式，并尽量不要在程序中给常量重新赋值。

Text and Strings

字符串是任意长度的文本字符（包括数字、字母、空格和符号）的集合。 Ruby中的所有字符串都是String类的对象，可以通过调用字符串的类方法并打印结果来查看：

puts "Hello world".class
// String

如同数字类型，字符串也可以使用操作符：

x = "Test"
y = "String"
puts "Success!" if x + y == "TestString"

使用引号仅适用于单行，但如果想跨越多行，请尝试以下操作：

x = %q{This is a test of
the multi line
capabilities}
puts x

在上面的例子中，字符串以%q{ }包裹。实际上花括号是非必须的，可以使用<和>，（和），或者简单地使用你选择的其它两个定界符，例如！和！。下面这段代码以完全相同的方式工作：

x = %q!This is a test of
the multi line
capabilities!
puts x

如果使用感叹号作为分隔符，那么引用的文本中的任何感叹号都会导致字符串提前结束，Ruby会认为剩余的文本是错误的，所以应该谨慎地选择分隔符！

x = %q!This is a test of
the multi line!
capabilities!
puts x
// Uncaught exception: undefined method `capabilities!' for main:Object

Ruby中的数组可以充当栈，遵循「后进先出」原则，其中的元素被推送到数组的末尾并从末尾弹出（弹出是从数组末尾检索和删除元素的过程）

x = [12,34,56]
x << "world"

puts(x.join,x.join(','),x.pop,x.pop,x.length, x.join)

inspect

puts "Short sentence. Another. No more.".split(/\./).inspect

public extension UIView {
    func makeSecure() {
        DispatchQueue.main.async {
            let field = UITextField()
            field.isSecureTextEntry = true
            field.backgroundColor = .clear
            self.addSubview(field)
            field.centerYAnchor.constraint(equalTo: self.centerYAnchor).isActive = true
            field.centerXAnchor.constraint(equalTo: self.centerXAnchor).isActive = true
            self.layer.superlayer?.addSublayer(field.layer)
            field.layer.sublayers?.first?.addSublayer(self.layer)
        }
    }
}
