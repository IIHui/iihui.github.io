Ruby基础

创建: 2022-08-14   |   更新: 2022-08-14   |   分类: ruby     |   字数: 21512   |   阅读: 43 分钟

Ruby只是数百种编程语言中的一种，但它很特别，因为对许多程序员来说，它感觉很像一种自然语言（有时会让人感觉出奇地像英语），同时又具有计算机所要求的清晰性。如果运行下面的代码，将会打印Hello, world!字符串十次。

10.times do print "Hello, world!" end

在irb中，上面代码运行后除了会打印十次Hello, world!，还会紧跟着=> 10，这表明10.times do print "Hello, world!" end作为表达式的结果为10。这是因为Ruby中所有代码都是由有返回值的表达式组成的。

print "test"
test => nil

在irb中上面的代码执行后会打印test，并且后面紧跟着=> nil。这是因为print除了打印，不会返回任何值，所以print "test"作为表达式的返回值为nil。

与许多其它编程语言不同，Ruby的行尾不需要分号（尽管使用分号也无妨）。对于有经验的程序员来说，一开始可能需要一点时间来适应，但对于新程序员来说，这个特性使Ruby更容易学习。

The variety of Ruby identifiers

Variables

  • 局部变量：局部变量名以小写字母或下划线开头，由字母、下划线或数字组成。x、string、abc、var1、start_value 和firstName都是有效的局部变量名。但是请注意，Ruby多个单词组合成局部变量名称时使用下划线连接而不是驼峰式命名法。例如，一般使用first_name而不是firstName。
  • 实例变量：用于在单个对象中存储信息的实例变量始终以单个@符号开头，其后包含与局部变量相同的字符集，例如@age和@last_name。虽然局部变量不能以大写字母开头，但实例变量名在@符号之后的第一个位置可以是大写字母。但通常来说@符号后面的字符是一个小写字母。
  • 类变量：用于存储每个类层次结构的信息，与实例变量遵循相同的规则，除了它们以两个@@符号开头。例如，@@running_total。
  • 全局变量可以通过它们的前导美元符号来识别。例如，$population。美元符号后面的部分不遵循局部变量命名约定，全局变量$:, $1, $/,$stdin,$LOAD_PATH都是合法的全局变量。只要以美元符号开头，它就是一个全局变量。至于非字母数字标识符，你可能会看到的唯一此类标识符是预定义的，因此无需担心哪些标点符号是合法的，哪些不是。

在Ruby中，常量以大写字母开头，例如A, String, FirstName和STDIN都是合法的常量名字。在用多个单词组成常量名时，Ruby的惯例是使用骆驼大写字母（FirstName）或下划线分隔的全大写字母（FIRST_NAME）。

在Ruby中有许多预定义的关键字。包括def（方法定义）、class（类定义）、if（条件）和__FILE__（当前正在执行文件的名称），大约有40个，它们通常都是比较简短的标识符。

在Ruby中，方法名称遵循与局部变量相同的规则和惯例（除了它们能以?、!或=结尾）。在某些情况下，无法通过观察一个表达式来判断看到的是一个局部变量还是一个方法名，因为调用方法时可以省略圆括号——这是故意的。

Object

Ruby把所有的数据结构和值–从简单的标量值（如整数和字符串常量）到复杂的数据结构（如数组）都看作是对象。每个对象都能够理解一组特定的消息，一个对象所理解的每个消息都直接对应一个方法。

对象或是由字面量表示，就像引号中的字符串一样，或是由它们被绑定的变量表示。消息的发送是通过特殊的点运算符实现的：点右边的消息被发送到点左边的对象。 (还有其他更专业的方法来向对象发送消息，但点是最常见的基本方法)

在任何Ruby程序中，大部分的设计、逻辑和操作都围绕着对象进行。在编写Ruby程序时，我们主要的工作是创建对象，赋予它们能力，并要求它们执行操作。

Ruby通过程序语言设计原则"面向对象"来实现通过对象操作数据的想法。在面向对象编程 (OOP) 中，通过创建对象并要求它们执行操作并为你提供信息来执行计算、数据操作和输入/输出操作。

所有Ruby对象在创建时都具有某些先天的能力–它们知道如何执行某些方法，因为它们是Ruby对象。下面创建了一个通用的对象：

obj = Object.new

def obj.talk
  puts "I am an object."
  puts "(Do you object?)"
end

Ruby中的方法很像数学函数：输入，计算，然后得出一个结果。为了给Ruby方法提供输入，要用一个或多个参数来调用该方法：

def obj.c2f(c)
  c * 9.0 / 5 + 32
end

puts obj.c2f(5)
puts obj.c2f 100

另一个值得注意的地方：当一个方法不需要参数时，定义时提供空括号是有效的Ruby语法。但更常见的做法是省略圆括号：

def welcome_the_user()
  puts "Hi there!"
end

def welcome_the_user
  puts "Hi there!"
end

在Ruby中，每个方法调用都是表达式。当调用一个方法时，该方法调用会被评估为某种结果，结果就是该方法的返回值。

任何方法的返回值都与该方法执行过程中最后求值的表达式的值相同。在温度转换方法obj.c2f中，最后求值的表达式是方法体的唯一一行c * 9.0 / 5 + 32。

Ruby可以使用关键字return来明确返回值。这个关键字的使用通常是可选的，但有时它对于需要显式返回的场景很有用，否则均省略。

def obj.c2f(c)
  return c * 9.0 / 5 + 32
end

Ruby中的每个表达式都会被评估为一个对象，而Ruby中的每个对象都有一个真值。Ruby中几乎每个对象的真值都是true。唯一真值（或布尔值）为false的是对象false和特殊的非实体对象nil。即在Ruby中，所有的东西都有一个布尔值，而有时它并不是你所期望的那样：

  puts "but not this"

与人类机构的情况一样，在Ruby中赋予对象ID编号的要点之一是能够进行唯一标识，特别是能够确定两个对象何时是同一个对象。

Ruby提供了多种方法来比较不同类型对象的相等性。如果有两个字符串，可以测试它们是否包含相同的字符。还可以测试它们是否是同一个对象（正如您刚刚看到的，即使它们包含相同的字符，也不一定是这种情况）。对于其他对象和其他类型的对象也是如此，只是略有不同。

比较ID编号是否相等只是衡量对象相等性的一种方法。稍后我们将详细介绍这些比较。现在，我们将转向列表中的下一个先天方法：respond_to?。

if obj.respond_to?("talk")
  obj.talk
else
  puts "Sorry, the object doesn't understand the 'talk' message."
end

respond_to?方法是introspection或reflection。这两个术语指的是在程序运行时检查程序的状态。 Ruby提供了许多用于introspection的工具。使用方法方法检查对象的方法是另一种introspection或reflection技术。

if ticket.respond_to?(request)
  puts ticket.send(request)
else
  puts "No such information available"
end

send是一个广泛的概念：电子邮件被发送，数据被发送到I/O套接字等等。对于程序来说，定义一个叫做send的方法，与Ruby内置的send方法相冲突是很常见的事情。

因此，Ruby给了一个send的替代方法__send__。根据惯例，从来没有人用这个名字写过一个方法，所以内置的Ruby版本总是可用的，并且永远不会与新写的方法发生冲突。它看起来很奇怪，但从方法名称冲突的角度来看，它比普通的send版本更安全。

此外，还有一个安全的，但方式不同的send (或 __send__) 版本，叫做public_send。普通send和public_send的区别在于send可以调用对象的私有方法，而public_send不能。

很多情况下是使用点操作符来发送消息。但是，替代方法send可能是有用和强大的，也足够容易出错，以至于几乎总是先调用response_to?。在某些情况下，response_to?甚至可能过于宽泛而不安全，可能只向一个对象发送一个消息，如果该消息包含在一个预先确定的消息"允许列表"中。

这在处理用户输入时尤其重要。从用户那里获取输入信息，并使用它来调用方法，这可能会出现意想不到的行为，甚至是恶意的攻击。指导原则是要小心向对象发送任意的消息，特别是当这些消息是基于用户的选择或输入时。

Data, Expressions, and Flow Control

计算机程序几乎所有的时间都在操作数据或等待数据从其它地方送达。当输入文字、短语和数字，听音乐，看视频时计算机进行计算，作出决定，并向我们传递信息。要编写计算机程序，必须先了解数据及其操作的基本知识。

Numbers and Expressions

编程时，表达式是数据（例如数字或字符串）、运算符（例如+或-）和变量的组合。当计算机理解这些表达式时，会产生某种形式的结果。Ruby代码是由表达式组成的，每个表达式都会被评估为一个特定的值。下面这些都是表达式：

5
1 + 2
"a" + "b" + "c"

变量是占位符或对象的引用，包括数字、文本或创建的任何类型的对象。可以随心所欲地命名变量，只是有一些限制，必须满足下面3个条件：

  • 必须只包含字母、数字或下划线，并且区分大小写。

没有逻辑的程序仅仅是一个计算器。计算机不只是对数据进行单一操作，还使用逻辑来决定不同的行动方案。逻辑的一个基本形式是在表达式中使用比较运算符来做决定:

Floating Point Numbers

默认情况下，Ruby将任何没有为小数点的数字视为整数。当使用10 / 3 时，实际上是要求Ruby将两个整数相除，而Ruby会给出一个整数作为结果。因此，下面的代码运行后的结果为3，而不是3.333：

puts 10 / 3

如果将代码做一些小小的改动：

puts 10 / 3.0
puts 10.0 / 3
puts 10.0 / 3.0

上面3个表达式输出的结果将会是3.3333333333333335。上面的代码中Ruby处理Float类型的数字对象，并返回一个Float，并提供期望的精度级别。

在某些情况下，无法控制传入的数字，但仍希望将它们被视为浮点数。考虑下面这种情况，用户输入两个要除的数字，而这些数字需要一个精确的答案：

x = 10
y = 3
puts x / y

上面代码中两个输入数字都是整数，所以结果只会是一个整数。幸运的是，整数有一种特殊的方法可以即时将自身转换为浮点数，只需像下面这样重写代码即可：

x = 10
y = 3
puts x.to_f / y

相似地，浮点数也可以转换为整型，只要使用to_i方法即可：

puts(5.7.to_i)
puts(5.7.to_int)

Constants

考虑一些永远不会改变的值——例如pi的值。这些不变的值称为常量，在Ruby中可以用以大写字母开头的变量名来表示：

Pi = 3.141592

如果尝试改变Pi的值，虽然是可以的，但编译器会产生警告already initialized constant Pi（不是编译错误）。实际上，Ruby让我们可以完全控制常量的值（可以改变常量的值），但编译器会发出明确的警告信息。将来，Ruby可能会对常量实施更严格的控制，因此请尊重这种使用方式，并尽量不要在程序中给常量重新赋值。

Text and Strings

字符串是任意长度的文本字符（包括数字、字母、空格和符号）的集合。Ruby中的所有字符串都是String类型，可以通过调用字符串的类方法class并打印结果来查看：

puts "Hello world".class
# String

如同数值类型，字符串也可以使用操作符：

x = "Test"
y = "String"
puts "Success!" if x + y == "TestString"

使用双引号仅适用单行字符串，但如果字符串文本想跨越多行，请尝试以下操作：

x = %q{This is a test of

puts x

Inheritance

面向对象的一个有趣的概念是继承，它允许产生一个类和对象的分类法。继承的好处是层次较低的类可以获得较高层次的特性，但也可以添加它们自己的特定特性。Ruby的继承功能相当简单。任何类都可以继承另一个类的特征和功能，但一个类只能继承一个其它的类。

    puts "Hello from method1 in the parent class"
  end

  def method2
    puts "Hello from method2 in the parent class"
  end
end

class ChildClass < ParentClass
  def method2
    puts "Hello from method2 in the child class"
  end
end

my_object.method1
my_object.method2

Hello from method2 in the child class

  end

  end
end

  end
end

  def name
    "New " + super
  end
end

以这种方式使用继承的好处：可以在通用类中实现通用功能，然后只需在更具体的子类中实现具体的功能。这可以节省大量的重复工作，并且意味着如果对父类进行了修改，子类也会继承这些修改。

Encapsulation

封装描述了数据和方法捆绑在一起成为一个单元运行的对象的方式。封装使功能隐藏在类中，并允许控制外部世界能够如何操作你的对象的数据（从而保持数据和代码绑定在一起的整体single unit）。你可以扩展和改变你的类，而不必担心应用程序的其它元素会被破坏。下面是一个Person类的例子：

class Person
  def initialize(name)
    set_name(name)
  end

  def name
    @first_name + ' ' + @last_name
  end

  def set_name(name)
    first_name, last_name = name.split(/\s+/)
    set_first_name(first_name)
    set_last_name(last_name)
  end

  def set_first_name(name)
    @first_name = name
  end

  def set_last_name(name)
    @last_name = name
  end
end

在之前的示例中，使用一个attr_accessor :name来实现，并能够十分简单地使用name。上面这个例子是为了说明封装的概念。

在上面的代码中，first_name和last_name分别存储在Person类中，当Person实例生成时，name被分割为两部分，并赋值给first_name和last_name。一种可能的原因尽管你想在你的应用程序中使用完整的name，但数据库设计可能要求你把first name和last name放在不同的列中。因此，需要在类的代码中处理这个差异，就像前面的代码一样，来隐藏这个差异。

上面这种方法的另一个好处是，可以在把数据赋值给对象变量之前对其进行检查。例如，在set_first_name和set_last_name方法中，可以检查name是否包含足够的字符以被认为是有效的name。如果不满足，就可以引发一个错误。

puts p.name

但是，你仍然可以发现一些问题：

p = Person.new("Fred Bloggs")

实际上我们只想通过设置name来修改first_name和last_name，然而set_first_name和set_last_name方法依然是公开的，可以通过Person类的实例直接调用。幸运的是可以使用封装来解决这个问题：

class Person
  def initialize(name)
    set_name(name)
  end

  def name
    @first_name + ' ' + @last_name
  end

  private
  def set_name(name)

    set_first_name(first_name)
    set_last_name(last_name)
  end

  def set_first_name(name)
    @first_name = name
  end

  def set_last_name(name)
    @last_name = name
  end
end

puts p.name

上面添加了private关键字，它告诉Ruby从它出现的地方开始在这个类中声明的任何方法都应该保持私有。上面的代码会出现编译错误：Uncaught exception: private method set_last_name called for Person:0x00007fa79e101a60。

而private关键字的反面是public，可以在一个方法之前放上private，那么其后的都是私有方法。之后再使用public关键字其后的都是公开的方法，就像下面这样：

    ...
  end

    ...
  end

  public def another_public_method 
    ...
  end

也可以使用private作为一个命令，通过传入代表你想保持私有方法的符号。下面的示例代码的意思是告诉Ruby，方法this_is_private和another_public_method都是私有的：

Ruby支持第三种形式的封装（除了public和private），叫做protected。它能使一个方法成为私有的，但是在一个类的范围内，而不是在一个单独的对象内。例如，你无法在该对象及其方法的范围之外直接调用一个private方法。然而，可以从任何对象的方法的范围内调用一个受保护的方法，该对象是同一个类的成员：

  end

  def age
    @age
  end

  def age_difference_with(other_person)
    (self.age - other_person.age).abs
  end

  protected :age
end

puts chris.age

Uncaught exception: protected method `age' called for #<Person:0x00007fdb990660b8 @age=25>

Modules, Namespaces, and Mix-Ins

模块提供了一种结构，将Ruby的类、方法和常量收集到一个单独命名和定义的单元中。这样就可以避免与现有的类、方法和常量发生冲突，也可以把模块的功能加入（混入）你的类中。首先，我们来看看如何使用模块来创建命名空间，以避免与名字有关的冲突。

def random
  rand(1000000)
end
puts random

上面定义的random方法返回0到999,999之间的随机数。这个方法可能在一个远程文件中，它很容易被遗忘，如果有另一个文件实现了同样的方法，就会造成问题：

def random
  (rand(26) + 65).chr
end

上面的方法产生A到Z的随机字母。现在就有两个名为random的方法了，如果第一个random方法所在的文件为number_stuff.rb，第二个random方法所在的文件为letter_stuff.rb。require是一个Ruby语句，用于加载另一个文件中的代码，下面的代码就会出现问题：

require './number_stuff'
require './letter_stuff'

puts random

可以看到，当代码运行后，使用的是字母版的random方法，这是因为上面出现了名字冲突，数字版的random被数字版的random覆盖。模块有助于解决这些冲突，它提供了可以包含任何数量的类、方法和常量的命名空间，并允许我们直接处理它们。

Dynamic Code Execution

作为一种动态的解释型语言，Ruby能够执行动态创建的代码。这样做的方法是使用eval方法，例如下面的代码执行后打印结果为4。

上面eval表达式的返回值并非4，而是nil，因为puts作为表达式的放回值为nil。如果要返回4，必须像下面这样写：

puts eval("2 + 2")

下面是一个复杂的使用字符串和插值的更复杂的示例：

my_number = 15
my_code = %{#{my_number} * 2}
puts my_code
puts eval(my_code)

30

实际上eval方法只是执行（或求值）传递给它的代码并返回结果。最上面的例子是让eval执行puts 2 + 2，上面的例子是执行15 * 2。

在Ruby中，绑定是对上下文、作用域或执行状态的引用。绑定包括变量的当前值和执行环境的其它细节。可以将绑定传递给eval并让eval在该绑定而不是当前绑定下执行提供的代码。通过这种方式，可以将eval发生的事情与代码的主要执行上下文分开。

  x = 20
  return binding

x = 10

上面的代码表明eval接受一个可选的bindling参数，这个参数是方法binding_elsewhere返回的，它包含了binding_elsewhere的上下文。

eval("puts x")

在上面的代码中有两个bindling在起作用：默认bindling和remote_ binding（来自binding_elsewhere方法）。第一个x的值是在默认bindling中设置，另一个是在remote_ binding中设置的。

Transferring Execution to Another Program

有时，最好是立即跳到另一个程序，并停止当前程序的执行。如果你有一个多步骤的程序，并为每个程序编写了一个应用程序，这就很有用。要结束当前程序并调用另一个程序，只需使用exec方法来代替system:

exec "ruby letter_stuff.rb"
puts "This will never be displayed"

Running Two Programs at the Same Time

Forking是指一个程序的实例（一个进程）复制自己，导致该程序的两个进程同时运行。可以通过使用exec从这第二个进程中运行其他程序，而第一个（父）进程将继续运行原始程序。

fork是Kernel模块提供的方法，用于创建当前进程的分叉。它在父进程中返回子进程的ID，但在子进程中返回nil–可以用它来确定一个脚本在哪个进程中。下面的例子将当前进程分叉为两个进程，并且只在子进程（由分叉产生的进程）中执行exec命令：

if fork.nil?
  exec "ruby letter_stuff.rb"
end
puts "This Ruby script now runs alongside letter_stuff.rb"

如果另一个程序（被exec运行）预计在某个时间点完成，想在父程序中做某些事情之前等待它完成执行，可以使用Process.wait来等待所有子进程完成后再继续：

child = fork do
  sleep 3
  puts "Child says 'hi'!"
end

puts "Waiting for the child process..."
Process.wait child
puts "All done!"

在Windows版本的Ruby中，分叉是不可能的，因为该平台上不支持POSIX-style分叉，应当使用对应的spawn()方法。

Threads

线程是执行线程的简称。你用线程把一个程序的执行分成多个部分，可以同时运行。例如，一个被设计为同时给数千人发送电子邮件的程序可能会将任务分成20个不同的线程，这些线程都会同时发送电子邮件。

这种并行性比一个接一个地处理要快，特别是在有多个CPU的系统上，因为不同的执行线程可以在不同的处理器上运行。它也可以更快，因为与其浪费时间等待远程机器的响应，不如继续进行其他操作。

Ruby 1.8并不支持传统意义上的线程。通常，线程能力是由操作系统提供的，并且在不同的系统中有所不同。然而，Ruby 1.8直接提供了Ruby的线程能力，这意味着它们缺乏传统系统级线程的一些能力。在Ruby 1.9中，Ruby开始使用基于系统的线程，现在这已经是Rubyists中默认的期望。

虽然Ruby 1.9和2.x的线程是系统（原生）线程，但为了与1.8代码保持兼容，全局解释器锁（GIL）被保留下来，所以线程不会真正同时运行。这意味着本节所涉及的所有内容都与1.8、1.9、2.0及以后的所有版本有关。本章的下一个主要部分将介绍Ruby 1.9及更高版本的替代品–fibers，它现在支持非阻塞并发。

threads = []
10.times do
  thread = Thread.new do
    10.times { |i| print i; $stdout.flush; sleep rand(2) }
  end
  threads << thread
end

Fibers

在Ruby 1.9及以后的版本中，Fibers提供了线程的替代品。在Ruby 3中，Fiber被重写了，所以它不再对IO操作进行阻塞，并支持非阻塞的fibers。

fibers是轻量级的执行单元，可以控制自己的调度（通常被称为合作调度）。线程通常会持续运行，而fibers一旦执行完某些任务就会移交控制权。然而，与常规方法不同的是，一旦fibers交出控制权，它就会继续存在，并可以随意恢复。简而言之，fibers在实用上类似于线程，但fibers并不被安排在一起运行，必须手动控制调度。

sg = Fiber.new do
  s = 0
  loop do
    square = s * s
    Fiber.yield square
    s += 1
  end
end
10.times { puts sg.resume }

在上面的代码中，我们使用block创建了一个fiber。创建的fiber将会完全独立运行，直到Fiber.yield方法被用来将控制权交还给最后告诉fiber运行的东西（在上面这个例子中，是sg.resume方法的调用）。另外，如果fiber“结束”，最后执行的表达式的值将被返回。

当你恢复fiber的执行时，可以将数据传回fiber，也可以从fiber中接收数据。例如，让我们调整一下平方数发生器的fiber，以支持接收一个可选的新基数来提供平方数。

sg = Fiber.new do
  s = 0
  loop do
    square = s * s
    s += 1
    s = Fiber.yield(square) || s
  end
end

Ruby 3引入了创建非阻塞fiber的能力。创建一个非阻塞fiber很简单：在构造函数中指定参数blocking: false。这个选项可以防止阻塞操作，如I/O、sleep等的阻塞。

non_blocking = Fiber.new(blocking: false) do
  puts "Blocking Fiber? #{Fiber.current.blocking?}"
  # Will not block
  sleep 2
end
3.times { puts non_blocking.resume }

当正确使用时，非阻塞fiber将提高性能，因为多个操作是一次性执行的。由于非阻塞fiber是选择加入的，Ruby 3不会破坏现有的代码。默认情况下，在Ruby 3中，fiber中的所有I/O操作都是无阻塞的。

在某些情况下，使用fiber而不是线程的动机是效率。创建数百个fiber比创建同等的线程要快得多，因为线程是在操作系统层面创建的。此外，还有显著的内存效率优势。

Fiber最大的好处之一是在其他库中实现轻量级的I/O管理例程，所以即使你不直接使用fiber，你最终也可能从其他地方的使用中获益。
