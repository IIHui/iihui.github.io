Failed to open macho file

在编写Swift脚本时，调试时出现了Failed to open macho file at xxx Too many levels of symbolic links报错信息，具体如下所示：

Failed to open macho file at /Applications/Xcode.app/Contents/Developer/Toolchains/XcodeDefault.xctoolchain/usr/bin/swift for reading: Too many levels of symbolic links

只需要按照参考方案添加如下代码即可去除该报错信息：

if ProcessInfo.processInfo.environment.keys.contains("OS_ACTIVITY_DT_MODE") {
    var env = ProcessInfo.processInfo.environment
    env["OS_ACTIVITY_DT_MODE"] = nil
    process.environment = env
}

can’t find gem cocoapods

使用rvm install 2.6.6命令安装了ruby 2.6.6版本。并使用 rvm use 2.6.6切换到这个版本后，再执行pod lib lint --verbose File.podspec命令时出现如下错误：

/System/Library/Frameworks/Ruby.framework/Versions/2.6/usr/lib/ruby/2.6.0/rubygems.rb:283:in `find_spec_for_exe': can't find gem cocoapods (>= 0.a) with executable pod (Gem::GemNotFoundException)
        from /System/Library/Frameworks/Ruby.framework/Versions/2.6/usr/lib/ruby/2.6.0/rubygems.rb:302:in `activate_bin_path'
        from /usr/local/bin/pod:23:in `<main>'

解决方案是执行sudo gem install cocoapods命令以安装cocoapods。

经排查是Xcode 14.0.1导致pod lib lint 失败，解决方案是将xcodebuild临时切到Xcode 13.4.1版本。可以在Xcode-->Preferences...-->Locations-->Command Line Tools中切换，修改完成后使用下面命令查看：

xcodebuild -version

# 结果
Xcode 13.4.1
Build version 13F100

如果xcodebuild -version命令的输出依然是原来的Xcode版本，就要检查一下是否已经在终端的配置文件中将DEVELOPER_DIR写死为某个版本。因为本机使用的是zsh，所以查看主目录下的.zshrc发现之前已经写死了版本，改成13.4.1版本即可。

# 修改前
export DEVELOPER_DIR="/Applications/XCode beta.app/Contents/Developer"

# 修改后
export DEVELOPER_DIR="/Applications/XCode 13.4.1.app/Contents/Developer"

rvm: command not found

在Intellij IDEA中运行Python脚本时，脚本中使用了rvm命令，出现如下所示报错。根源是当前工程未识别到rvm命令的路径，只需在环境变量中将该命令的路径加上即可：

/bin/sh: rvm: command not found

simulator listed or otherwise add one

使用pod lib lint命令检查podspec文件时出现了Could not find a ios simulator (valid values: )错误提示。根据报错信息提示，解决方案是在Xcode-->Window-->Devices and Simulators中至少添加一个模拟器。

ERROR | [iOS] unknown: Encountered an unknown error (Could not find a `ios` simulator (valid values: ). Ensure that Xcode -> Window -> Devices has at least one `ios` simulator listed or otherwise add one

Disable entire UIMenuController edit menu in WKWebView

如果想要在WKWebView上禁止长按弹出复制、粘贴等选项，可以使用如下方式：

override var canBecomeFirstResponder: Bool {
    return true
}

override var canResignFirstResponder: Bool {
    return false
}

 public override func canPerformAction(_ action: Selector, withSender sender: Any?) -> Bool {
     return false
 }

NotificationCenter.default.addObserver(self, selector: #selector(willShowMenu(_:)), name: UIMenuController.willShowMenuNotification, object: nil)

@objc private func willShowMenu(_ notification: NSNotification) {
     UIMenuController.shared.isMenuVisible = false
     UIMenuController.shared.update()
}
