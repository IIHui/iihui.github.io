Swift访问权限

时间： 2019-01-09   |   分类： code     |   阅读： 776 字 ~2分钟

访问权限用于限制部分代码被其它源文件或者模块访问，目的是隐藏某些代码的实现细节。在Swift中可以设置类型(类、结构和枚举)的访问权限，也可以设置属性和方法的访问权限。Swift提供了5个访问级别，这些访问级别以模块和源文件为区分粒度。

模块和源文件

模块

模块是指独立的代码单元，在iOS中框架或应用程序就是一个独立模块。在一个模块中可以用import关键字导入另一个模块。

源文件

源文件即Swift中的源代码文件，它通常属于一个模块，即一个框架（Framework）或者应用程序。

访问级别

open 与 pulic

由open和public修饰的实体可以被模块内的任何文件访问，也可以被主动导入这个模块的主模块中的源文件使用。当要定义框架的公开接口时，通常使用open或者public。

区别

虽然open和public都有最高的访问级别，但open访问权限只适用于修饰class和class的成员，它与public有明显的不同：

  • 由public修饰的class，或被其他更严格的访问级别修饰，这个class只能在当前模块内被继承。

  • 由public修饰的class成员，或被其他更严格的访问级别修饰，这些成员只能被当前模块内的子类重写。

  • 由open修饰的class，不仅能在当前模块内被继承，而且也可以在导入了这个模块的另一个模块内被继承。

  • 由open修饰的class成员，不仅能被当前模块内的子类重写，而且也可以被导入了这个模块的另一个模块内的子类重写。

internal

由internal修饰的实体可以被模块内的任何文件访问，模块之外的其它文件不能访问。这是所有实体的默认权限。

fileprivate

由fileprivate修饰的实体可以在当前文件内被访问。若某个功能只要在文件内使用，则可以使用这个访问符修饰。

private

由private修饰的实体只能被实体所在的声明内部访问，当实现细节只在声明内部使用时使用private修饰。
