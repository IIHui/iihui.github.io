Chapter 1

单个词的变量名可能很好，但有时希望使用多个单词来使变量或常量名有更好的可读性。当组合多个单词形成变量或常量名时，通常使用camelCase命名法：

var ageOfPet: Int = 8 
var weightInKilograms: Double = 13.48 
var directionToTurn: String = "Right"

除了将相关项目进行物理分组，如函数和变量分别放在一起，也可以通过添加//MARK: 注释使得搜索代码块变得更容易。放置一个//MARK: 注释，后面紧跟描述性文字，那么通过Xcode的下拉菜单可以很容易从一段代码跳转到另一段，如下所示：

在SwiftUI中设计用户界面时，通常会创建一个视图（如Text），然后应用对应的修改器。如果需要反复使用相同的修改器，可以为多个视图重复这些修改器代码。但是，重复使用同样的修改器代码会使代码看起来冗长：

Text("This is the first line")
        .font(.title)
        .foregroundColor(.yellow)
        .background(Color.blue)
        .cornerRadius(6)
        .padding()

Text("Second line here")
        .font(.title)
        .foregroundColor(.yellow)
        .background(Color.blue)
        .cornerRadius(6)
        .padding()

除了用重复的代码占用空间，还会带来另一个问题。如果想改变一个或多个修改器，如改变字体大小或背景颜色。使用重复的修改器，需要修改每一个副本，增加了忘记修改一个或多个副本的概率。

一个更好的解决方案是将常用的修改器组一起存储在一个定义为ViewModifier的独立struct中。然后可以将这个ViewModifier结构应用于多个view。现在如果需要改变这些修改器，可以在一个地方改变它们，而不是在多个地方。

struct MyStyle: ViewModifier {
    func body(content: Content) -> some View {
        content
                .font(.title)
                .foregroundColor(.yellow)
                .background(Color.blue)
                .cornerRadius(6)
                .padding()
    }
}

上面定义的这个ViewModifier结构包含了所有的修改器，然后可以通过使用.modifier跟着结构名称来应用于任何视图，像下面这样:

struct ContentView: View {

    var body: some View {
        VStack {
            Text("This is the first line")
                    .modifier(MyStyle()) 
            Text("Second line here")
                    .modifier(MyStyle())
        }
    }
}

Closures

把函数改写成闭包的一种方法是去掉func关键字和函数名，然后把其余的代码用大括号括起来，像下面这样：

let y = { (x: Int) -> Int in
    return x * 2
}

还可以将上面的方法简写为如下形式：

let z = {x in return x * 2}

let w = {x in x * 2}

let v = {$0 * 2}

当声明一个函数时，需要显式定义每个参数的类型：

func addNumbers(x: Int, y: Int) -> Int {
    return x + y
}

使用闭包时，需要把所有的参数放在括号里。在许多情况下，不需要定义每个参数的数据类型，因为Swift可以根据返回值的数据类型来推断类型。例如，如果返回值数据类型是整数，Swift就会推断出传递的参数也必是整型，例如：

{(x, y) -> Int in return x + y}

但是，如果有任何不明确的地方，就必须明确地定义参数的数据类型：

{(x: Int, y: Int) in return x + y} 
{(x: Int, y: Int) in x + y} 
{$0 as Int + $1 as Int}

Understanding Value Capturing

当在一个函数中声明变量和常量时，它们只能在该函数中被访问。然而，当在一个函数之外声明一个变量或常量时，该函数可以访问该值：

也许闭包最通用的用法是把它们当作数据块，可以像使用任何固定值一样使用它们。这意味着可以将一个闭包作为一个函数（或另一个闭包）的参数来传递，将闭包存储在像数组这样的数据结构中，或者将闭包分配给一个变量。

let addNumbers1 = { (x, y) -> Int in
    return x + y
}
let addNumbers2 = { (x: Int, y: Int) in
    return x + y
}
let addNumbers3 = { (x: Int, y: Int) in
    x + y
}
let addNumbers4 = {
    $0 as Int + $1 as Int
}

Chapter 1

在新建的SwiftUI工程中，主文件是ContentView.swift，它里面包含了一个应用程序定义用户界面的Swift代码：

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundColor(.accentColor)
            Text("Hello, world!")
        }
        .padding()
    }
}

struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        ContentView()
    }
}

上面代码中，import SwiftUI可让应用程序使用SwiftUI框架来设计用户界面。在文件ContentView.swift中自动生成了两个View：

ContentView：一个视图结构体在屏幕上显示单个视图，SwiftUI 一次只能在屏幕上显示一个视图。创建 SwiftUI iOS 应用程序时，默认视图是一个垂直堆栈 (VStack)，它里面包含一个图像视图和一个文本视图。这个图像视图显示一个地球仪图标，文本视图在屏幕上显示 Hello, world!。

ContentView_Previews：是一个PreviewProvider 结构体，实际上用于在Canvas面板中显示定义的用户界面，该面板位于 Swift代码面板的右侧：

当编辑面板和画布面板并排出现时，编辑面板中的任何修改都会显示在预览面板中，反之亦然。点击右上角的编辑器选项可以关闭画布面板。

画布面板有两个作用。首先，它可以让我们看到用户界面在模拟器上是什么样子的。除了可以在不同的模拟器（如不同的iPhone或 iPad型号）之间切换外，画布（Canvas） 面板还可以让我们选择不同的设置，如明暗模式和动态类型。更改iOS设置可以让我们看到用户界面在不同显示选项下的效果。

  2. Xcode还提供了第二种方式切换画布面板的模拟器，即使用Inspector面板切换。首先将光标切到ContentView_Previews中的ContentView，然后在出现的Inspector面板点击Device，选择即可：

当选择好了不同的模拟器设备，则可以通过单击画布面板右下角出现的不同图标来更改画布面板的放大倍率，如下图所示：

  • Zoom to Fit: 使iOS设备全部可见。
  • Zoom In：放大iOS设备尺寸。

在画布面板中选择要模拟的iOS设备后，还可以自定义三种方法来更改用户界面的外观：

  • Color Scheme：显示浅色或深色模式。
  • Orientation：显示纵向或横向模式。
  • Dynamic Type：改变文字大小。

Manipulating the Xcode Panes

Xcode的4个面板，导航，编辑，画布和检查器有不同的用途。导航器面板显示有关项目的信息，例如组成项目的所有文件的名称。在编辑器面板中，可以编写和编辑Swift代码。画布面板用于查看和测试由Swift代码定义的用户界面，检查器面板显示当前选定对象的相关信息。

对于导航或检查器面板，可以使用如下方式进行显示和隐藏:

  • 选择View--->Navigators/Inspectors--->Hide/Show Navigator/ Inspector。
  • 单击显示（或隐藏）导航器（或检查器）面板图标。

Chapter 2

每个应用程序都需要一个用户界面。SwiftUI的基本思想是使用称为"views“的构件来创建用户界面。视图在用户界面上显示单个项目，如文本、图像或按钮，如下图所示：

SwiftUI的一个限制是一次只能在屏幕上显示一个视图。为了摆脱这一限制，SwiftUI提供了一种名为"stack"的功能。堆栈被视为单个视图，但允许组合或堆叠多达十个额外的视图。通过创建堆栈，可以在屏幕上显示多个视图。堆栈甚至还可以容纳其它堆栈，基本上可以让你在一个屏幕上显示任意多个视图。总共有3种类型的堆栈：

  • VStack：将视图排列在另一个视图的上方和下方。
  • HStack：并排排列视图的水平堆栈。
  • ZStack：将视图直接叠加在一起的堆栈。

一个堆栈可视为一个视图。通过使用水平（HStack）或垂直（VStack）堆栈，可以在堆栈内添加多达十个视图。为了获得更大的灵活性，可以将堆栈嵌入堆栈内部，根据需要显示多个视图。

一个堆栈最多只能包含十个视图。如果尝试在堆栈中存储11个或更多视图，Xcode将显示错误信息并拒绝运行你的程序。

在SwiftUI中创建用户界面时，有三个选项：

  • 在编辑面板书写Swift代码。
  • 将视图（如按钮）拖放到编辑器窗格中的Swift代码中。
  • 将视图（如按钮）拖放到画布面板中。

在编辑面板编写Swift代码来设计用户界面是最快、最灵活的方法，但需要花费时间并熟悉不同的选项。为了让键 Swift代码定义用户界面视图变得更容易，Xcode 在识别到试图键入的内容时会弹出一个选项菜单。选择一个选项并按下Return键，就可以快速创建用户界面视图。

如果对设置用户界面的各种选项不熟悉，可以使用Library Window，它列出了各种用户界面视图。一旦打开了Library Window，可以使用：

  • 将用户界面视图从库窗口拖放到编辑器面板中。
  • 单击"可选"图标，然后将用户界面视图拖放到画布面板中的模拟iOS设备用户界面上。

无论如何更改用户界面，Xcode都会保持编辑器面板和画布面板之间的所有更改同步。这意味着当在编辑器面板中输入Swift代码时，画布面板会立即显示这种更改。当将用户界面视图拖放到画布面板中时，Xcode会自动将 Swift代码添加到编辑器窗格中。

画布面板显示用户界面，但如果想测试应用程序，则有两种选择：

  • 单击运行按钮或选择产品 ➤ 运行打开模拟器。
  • 在画布面板的底部点击Live按钮。

要了解SwiftUI如何创建可响应用户的简单用户界面，请按以下步骤操作：

  • 创建一个新的iOS应用程序项目，确保界面为SwiftUI，并给它起一个描述性的名称（如 SwiftExample）。

  • 单击导航器窗格中的ContentView文件，编辑器窗格会显示ContentView文件的内容。

  • 选择编辑器 ➤ 画布。(如果画布选项前已出现复选标记，则跳过此步骤）。这将打开画布，以便预览用户界面。

  • 删除文本命令中的文本"Hello, world!"，然后键入以下内容，使ContentView结构如下所示：

    struct ContentView: View {
    @State private var message = true
    var body: some View {
        VStack {
            Toggle(isOn: $message) {
                Text("Toggle message on/off")
            }
            if message { Text ("Here's a secret message!") }
        }
        .padding()
    }
}

    上面的Swift代码会在屏幕上显示一个拨动开关，但除非在模拟器（模拟iPhone或iPad等iOS设备）中运行应用程序或单击实时图标进行测试，否则无法看到它的工作状态。在大多数情况下，“实时"图标能更快地在画布面板中查看和测试用户界面。

  • 单击实时图标打开实时预览。单击切换时，请注意信息的出现和消失。

单击运行按钮打开模拟器，重复前面的步骤，也可以完成测试。在模拟器或画布面板中测试应用程序的方法相同，主要区别在于画布面板通常使用起来更快。

VStack中的第一个视图是Toggle，它使用文本视图在Toggle左侧显示以下文本： Toggle message on/off。当Toggle打开时，message状态变量的值将变为true。当Toggle关闭时，message状态变量将存储一个false值。

在Toggle下面，有一个if语句。如果 message值为true，则会显示一个文本视图，显示“Here’s a secret message!" 如果message状态变量为false，则不显示文本视图。

Modifying the User Interface with the Inspector Pane

在SwiftUI中创建用户界面通常包括两个步骤。首先，需要安排各种用户界面视图在屏幕上的显示方式。接下来，需要通过更改大小、颜色或位置来定制每个用户界面视图。

要自定义用户界面视图，需要添加modifiers。Xcode 提供了两种添加modifiers的方法：

  • 直接在编辑器面板中键入modifiers。

  • 单击要修改的视图（在编辑器面板中或单击可选图标后，再单击画布面板中的视图）。然后从检查器窗格中选择修改器。

直接在编辑器面板中键入修改器需要知道要使用的修改器名称。随着经验的积累，直接在Swift代码中键入修饰符的速度会越来越快。不过，刚开始使用时，可能不知道哪些修饰符可用。这时，可能更喜欢使用检查器面板。

首先，选择要修改的用户界面视图。可以将光标移到定义该用户界面视图的Swift代码中，或者单击可选图标，然后单击画布面板中的用户界面视图。选择用户界面视图后，检查器面板会显示该视图的修改器。

检查器面板显示最常用的修改器，例如选择文本的字体、对齐方式或颜色。单击检查器面板底部的添加修改器按钮，可以查看其它修改器列表，如下图所示：

当从添加修改器列表中看到想要使用的修改器时，点击该修改器。Xcode将在Inspector面板中显示该修改器。如果不想再看到已添加到检查器面板中的修改器，请单击该修改器右上角的删除按钮。

需要注意的是，应用修饰符的顺序会产生不同的效果。下面的文本视图有两个修饰符，背景和填充：

Text("Here's a secret message!").background(Color.yellow).padding()

上面代码中，会为文本视图添加黄色背景色，然后在文本视图周围添加填充（空间）。假设将这些修改器调换一下，效果如下：

Text("Here's a secret message!").padding().background(Color.yellow)

上面代码首先在文本视图周围添加填充（空间），然后为背景着色。由于背景现在包含了添加的空间，因此背景颜色也填充了文本视图周围的空间。

Chapter 3

在用户界面上放置单个视图时，无论屏幕是小巧的 iPhone屏幕还是大得多的iPad屏幕，SwiftUI都会将其置于屏幕中央。当需要在用户界面上添加更多视图时，SwiftUI只需将这些视图相互叠加或并排显示，具体取决于是以垂直还是水平堆叠的方式排列这些视图。

但是，添加到用户界面的视图越多，相邻视图就会显得越拥挤。为了解决这个问题，SwiftUI提供了几种在用户界面上放置视图的方法：

  • 使用padding修饰器。
  • 在stack中定义spacing。
  • 在视图之间使用spacer。
  • 定义偏移或定位位置。

可以使用一种或多种不同的方法在用户界面上排列视图，使它们准确地显示在你想要的位置。最重要的是，这些定位方法适用于SwiftUI中的所有用户界面视图。

Using the Padding Modifier

padding修饰符可在用户界面视图周围添加空间。默认情况下，padding修饰符会在视图的顶部、底部、前方（左侧）和后方（右侧）添加空间。要使用padding修饰符，只需在任何视图后添加以下内容即可:

.padding()

填充有两个作用。首先，它可以在视图周围增加空间，从而改变背景，可以在背景上添加颜色，使视图更大更容易看清。其次，填充修饰器会将相邻视图推远，使相邻视图更容易看清，消除拥挤。

最简单的填充修改器会在视图的四边添加16个点间距。如果添加一个数字，就可以定义所需的精确间距，例如 .padding(45) 或 .padding (3):

Text("Default padding of 16 points").padding().background(Color.yellow)
Text("Default padding of 45 points").padding(45).background(Color.yellow)
Text("Default padding of 3 points").padding(3).background(Color.yellow)

请注意，填充修饰器会在所有边上添加间距，可以在一个或多个特定区域定义间距：

  • .top
  • .bottom
  • .vertical(top and bottom)
  • .leading(left)
  • .trailing(right)
  • .horizontal (trailing and leading)

如果没有指定值，那么SwiftUI会增加默认16个点的空间。要同时定义添加填充的区域和特定间距，必须先定义添加填充的区域，然后再定义特定值：

.padding(.top, 30)

如果要为两个或三个区域添加间距，可以像下面这样在方括号中定义这些区域，并在后面加上可选的间距值：

.padding([.top, .leading])
.padding([.top, .leading] , 30)

与其使用固定数值来定义填充的间距，还不如使用一个表示数值的变量，就像下面这样：

var distanceSize = 15.0
.padding(CGFloat(distanceSize))

需要注意的是，.padding修饰符专门将任何数值转换为CGFloat数据类型。通过在.padding修饰符中使用数值变量定义间距，代码可以通过改变该变量的值来改变.padding修饰符的间距。

Defining Spacing Within a Stack

.padding()修饰符可以方便地将不同视图分开。如果没有填充，堆栈中的多个视图就会挤在一起。如下图所示，通过为堆栈中的每个视图添加填充，可以将它们分开，这样每个视图就更容易看清了。

虽然填充可以让每个视图更容易看清，但可能希望对堆栈内视图之间的间距有更多控制。为此，可以在定义堆栈时定义一个间距值，例如：

VStack (spacing: 40) { }

Aligning Views Within a Stack

创建堆栈（VStack或HStack）时，无论是否定义了间距，都可以选择定义对齐方式。如果在堆栈中同时定义对齐方式和间距，则必须先定义对齐方式，后定义间距：

VStack (alignment: .leading) 
VStack (alignment: .leading, spacing: 24)

如下图所示，使用VStacks有三种方法对齐视图：

  • .leading (left)
  • .center (default setting if no other alignment options are chosen)
  • .trailing (right)
