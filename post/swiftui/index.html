Chapter 1

单个词的变量名可能很好，但有时希望使用多个单词来使变量或常量名有更好的可读性。当组合多个单词形成变量或常量名时，通常使用camelCase命名法：

var ageOfPet: Int = 8 
var weightInKilograms: Double = 13.48 
var directionToTurn: String = "Right"

除了将相关项目进行物理分组，如函数和变量分别放在一起，也可以通过添加//MARK: 注释使得搜索代码块变得更容易。放置一个//MARK: 注释，后面紧跟描述性文字，那么通过Xcode的下拉菜单可以很容易从一段代码跳转到另一段，如下所示：

在SwiftUI中设计用户界面时，通常会创建一个视图（如Text），然后应用对应的修改器。如果需要反复使用相同的修改器，可以为多个视图重复这些修改器代码。但是，重复使用同样的修改器代码会使代码看起来冗长：

Text("This is the first line")
        .font(.title)
        .foregroundColor(.yellow)
        .background(Color.blue)
        .cornerRadius(6)
        .padding()

Text("Second line here")
        .font(.title)
        .foregroundColor(.yellow)
        .background(Color.blue)
        .cornerRadius(6)
        .padding()

除了用重复的代码占用空间，还会带来另一个问题。如果想改变一个或多个修改器，如改变字体大小或背景颜色。使用重复的修改器，需要修改每一个副本，增加了忘记修改一个或多个副本的概率。

一个更好的解决方案是将常用的修改器组一起存储在一个定义为ViewModifier的独立struct中。然后可以将这个ViewModifier结构应用于多个view。现在如果需要改变这些修改器，可以在一个地方改变它们，而不是在多个地方。

struct MyStyle: ViewModifier {
    func body(content: Content) -> some View {
        content
                .font(.title)
                .foregroundColor(.yellow)
                .background(Color.blue)
                .cornerRadius(6)
                .padding()
    }
}

上面定义的这个ViewModifier结构包含了所有的修改器，然后可以通过使用.modifier跟着结构名称来应用于任何视图，像下面这样:

struct ContentView: View {

    var body: some View {
        VStack {
            Text("This is the first line")
                    .modifier(MyStyle()) 
            Text("Second line here")
                    .modifier(MyStyle())
        }
    }
}

Closures

把函数改写成闭包的一种方法是去掉func关键字和函数名，然后把其余的代码用大括号括起来，像下面这样：

let y = { (x: Int) -> Int in
    return x * 2
}

还可以将上面的方法简写为如下形式：

let z = {x in return x * 2}

let w = {x in x * 2}

let v = {$0 * 2}

当声明一个函数时，需要显式定义每个参数的类型：

func addNumbers(x: Int, y: Int) -> Int {
    return x + y
}

使用闭包时，需要把所有的参数放在括号里。在许多情况下，不需要定义每个参数的数据类型，因为Swift可以根据返回值的数据类型来推断类型。例如，如果返回值数据类型是整数，Swift就会推断出传递的参数也必是整型，例如：

{(x, y) -> Int in return x + y}

但是，如果有任何不明确的地方，就必须明确地定义参数的数据类型：

{(x: Int, y: Int) in return x + y} 
{(x: Int, y: Int) in x + y} 
{$0 as Int + $1 as Int}

Understanding Value Capturing

当在一个函数中声明变量和常量时，它们只能在该函数中被访问。然而，当在一个函数之外声明一个变量或常量时，该函数可以访问该值：

也许闭包最通用的用法是把它们当作数据块，可以像使用任何固定值一样使用它们。这意味着可以将一个闭包作为一个函数（或另一个闭包）的参数来传递，将闭包存储在像数组这样的数据结构中，或者将闭包分配给一个变量。

let addNumbers1 = { (x, y) -> Int in
    return x + y
}
let addNumbers2 = { (x: Int, y: Int) in
    return x + y
}
let addNumbers3 = { (x: Int, y: Int) in
    x + y
}
let addNumbers4 = {
    $0 as Int + $1 as Int
}

Chapter 1

主SwiftUI文件被称为ContentView，它里面包含了一个应用程序定义用户界面的Swift 代码：

import SwiftUI

struct ContentView: View {
    var body: some View {
        VStack {
            Image(systemName: "globe")
                .imageScale(.large)
                .foregroundColor(.accentColor)
            Text("Hello, world!")
        }
        .padding()
    }
}

struct ContentView_Previews: PreviewProvider {
    static var previews: some View {
        ContentView()
    }
}

上面代码中，import SwiftUI行可让应用程序使用SwiftUI框架来设计用户界面。

ContentView：一个视图结构体在屏幕上显示单个视图。SwiftUI 一次只能在屏幕上显示一个视图。创建 SwiftUI iOS 应用程序时，默认视图是一个垂直堆栈 (VStack)，其中包含一个图像视图和一个文本视图。图像视图显示一个地球仪图标，文本视图在屏幕上显示 “Hello, world!"。

ContentView_Previews： PreviewProvider 结构体实际上是在Canvas 面板中显示用户界面，该面板位于 Swift代码面板的右侧：

当编辑面板和画布面板并排出现时，编辑面板中的任何修改都会显示在预览面板中，反之亦然。点击右上角的编辑器选项可以关闭画布面板。

画布面板有两个作用。首先，它可以让我们看到用户界面在模拟器上是什么样子的。除了可以在不同的模拟器（如不同的 iPhone或 iPad型号）之间切换外，Canvas 面板还可以让我们选择不同的设置，如明暗模式和动态类型。更改iOS设置可以让我们看到用户界面在不同显示选项下的效果。

Switching Between iOS Devices

画布面板能够模拟单个iOS设备，如iPhone 14 Pro或iPad Mini。这样就能看到用户界面如何适应不同iOS设备的屏幕尺寸。要使Canvas面板模拟不同的iOS设备，请按照以下步骤操作：

  1. 单击 Xcode窗口顶部当前显示的iOS设备名称，不同iOS设备的列表就会出现，然后选择想要的模拟器即可，如下图所示：

  2. Xcode还提供了第二种方式切换画布面板的模拟器，即使用Inspector面板切换。首先将光标切到ContentView_Previews中的ContentView，然后在出现的Inspector面板点击Device，选择即可：

当选择好了不同的模拟器设备，则可以通过单击 “画布 “面板右下角出现的不同图标来更改 “画布 “面板的放大倍率，如下图所示：

  • Zoom Out：缩小iOS设备尺寸。
  • Zoom to 100%: 以实际尺寸显示iOS设备（可能会截断iOS设备的某些部分，尤其是在模拟iPad或更大的iPhone型号时）。
  • Zoom to Fit: 使iOS设备全部可见。
  • Zoom In：放大iOS设备尺寸。

Changing the iOS Device Emulation Appearance

在画布面板中选择要模拟的iOS设备后，还可以自定义三种方法来更改用户界面的外观：

  • Color Scheme:显示浅色或深色模式。
  • Orientation：显示纵向或横向模式。
  • Dynamic Type：改变文字大小。

如果想要对比不同更改用户外观的各种选项，可以点击底部的变量图标，然后选择不同的变量，就可以看到各种选项值的对比图：

单击"实时"图标，再次在画布面板中查看iOS设备，如下图所示：

Selecting User Interface Objects

在用户界面上放置对象后，只能通过在该对象的Swift代码中移动光标来选择它们。如果希望使用鼠标点击用户界面上的对象，则必须点击画布面板底部的可选图标，如下图所示：

一旦开启可选图标，单击用户界面上的对象后，该对象周围会出现蓝色边框。通过这些蓝色边框，可以看到对象的大小。如果添加了背景颜色，还可以方便地查看对象背景的大小。实时图标可让我们与用户界面进行交互，以测试应用程序，可选图标可以方便地编辑用户界面。

Manipulating the Xcode Panes

Xcode的4个面板，导航，编辑，画布和检查器有不同的用途。导航器面板显示有关项目的信息，例如组成项目的所有文件的名称。在编辑器面板中，可以编写和编辑Swift代码。画布面板用于查看和测试由Swift代码定义的用户界面，检查器面板显示当前选定对象的相关信息。
