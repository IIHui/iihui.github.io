手动部署包

时间： 0001-01-01   |   分类： code     |   阅读： 538 字 ~2分钟

今天因流水线的自动部署故障，又加上项目中引入了百度语音合成，超过了100M，因此需要手动将打好的包传到后台服务器，然后设置这个包有效，这个过程总结为上传包–>手动新增–>设置有效。

上传包

首先将打好的包上传到\\99.12.115.36\c$\WebOA\App\ios目录下，如下图所示将030615这个包放到该目录

手动新增

然后进入到后台管理界面，选择APP版本管理–>APP版本管理:

接着在系统类型那一列选择移动办公iPhone版，再点击右边的查询按钮:

接着在查询结果界面点击新增：

接着在出现的界面选择发布类型，内部编译版本就填写.ipa文件对应的日期，公开版本号和内部编译版本保持一致即可，是否强制提示更新一般选择否，是否提示更新一般选择是，iOS整数到期时间一般填写一年后的当日，如下图所示：

然后APP包这一块先随便选择一个，使得出现安装地址，如下图所示：

接着在APP包这一项下选择请选择这一项，然后再在第二栏下填写.ipa文件的名字（不含后缀）：

最后修改安装地址，将最后一个%后面的内容替换为.ipa文件的名字（不含后缀），然后点击提交即可:

设置有效

然后回到管理界面，选中刚刚设置的那个版本，点击设置当前有效即可：
