清理文件

本地.git文件太大，可以使用git gc命令清理不必要的文件并优化本地存储库，其中gc是garbage collection的缩写:

git gc [--aggressive] [--auto] [--quiet] [--prune=<date> | --no-prune] [--force] [--keep-largest-pack]

大小写敏感

默认情况下git追踪的项目对文件名大小写是不敏感的，但是git提供了命令可以使得当前项目或者所有项目都能区分文件名的大小写。

当前项目

打开终端，利用cd命令，将目录切换到git仓库下，输入如下命令使其对文件名大小写敏感：

git config core.ignorecase false

所有项目

如果想要所有项目都对文件名大小写敏感，就需要进行全局设置。执行如下命令，改变git的全局设置即可：

git config --global core.ignorecase false
